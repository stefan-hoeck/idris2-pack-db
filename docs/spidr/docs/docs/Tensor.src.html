 377 | |||
 378 | ||| **Static indices**
 379 | |||
 380 | ||| Static indices are `Nat`s. For example, for
 381 | ||| ```
 382 | ||| x : Tensor [5, 6] S32
 383 | ||| x = tensor [[ 0,  1,  2,  3,  4,  5],
 384 | |||             [ 6,  7,  8,  9, 10, 11],
 385 | |||             [12, 13, 14, 15, 16, 17],
 386 | |||             [18, 19, 20, 21, 22, 23],
 387 | |||             [24, 25, 26, 27, 28, 29]]
 388 | ||| ```
 389 | ||| we can index as `slice [at 1] x` to get
 390 | ||| ```
 391 | ||| x : Tensor [6] S32
 392 | ||| x = tensor [6, 7, 8, 9, 10, 11]
 393 | ||| ```
 394 | ||| or we can slice as `slice [2.to 4] x` to get
 395 | ||| ```
 396 | ||| x : Tensor [2, 6] S32
 397 | ||| x = tensor [[12, 13, 14, 15, 16, 17],
 398 | |||             [18, 19, 20, 21, 22, 23]]
 399 | ||| ```
 400 | ||| Note that in `2.to 4`, the 2 is inclusive, and the 4 exclusive, so we return indices 2 and 3.
 401 | |||
 402 | ||| **Dynamic indices**
 403 | |||
 404 | ||| Dynamic indices are scalar `U64` values, and the API works slightly differently because we
 405 | ||| can't know the value of dynamic indices until the graph is executed. For indexing, with scalar
 406 | ||| `U64` index `i` in `slice [at i] x`, `i` is clamped to be a valid index into that dimension.
 407 | ||| For example, for `i = tensor 1`, `slice [at i] x` is
 408 | ||| ```
 409 | ||| x : Tensor [6] S32
 410 | ||| x = tensor [6, 7, 8, 9, 10, 11]
 411 | ||| ```
 412 | ||| as in the static case. However, for `i = tensor 10`, `slice [at i] x` returns the last row
 413 | ||| ```
 414 | ||| x : Tensor [6] S32
 415 | ||| x = tensor [24, 25, 26, 27, 28, 29]
 416 | ||| ```
 417 | ||| We can also slice by specifying a scalar `U64` start index, and a static size, as
 418 | ||| `slice [i.size 2] x` with `i = tensor 2` to get
 419 | ||| ```
 420 | ||| x : Tensor [2, 6] S32
 421 | ||| x = tensor [[12, 13, 14, 15, 16, 17],
 422 | |||             [18, 19, 20, 21, 22, 23]]
 423 | ||| ```
 424 | ||| For a given slice `size`, the dynamic start index is clamped such that we always get `size`
 425 | ||| elements along that axis. For example, `slice [i.size 2] x` with `i = tensor 4` is
 426 | ||| ```
 427 | ||| x : Tensor [2, 6] S32
 428 | ||| x = tensor [[18, 19, 20, 21, 22, 23],
 429 | |||             [24, 25, 26, 27, 28, 29]]
 430 | ||| ```
 431 | ||| which starts at index 3 rather than index 4.
 432 | |||
 433 | ||| **Mixed static, dynamic, slicing and indexing**
 434 | |||
 435 | ||| Each axis can only be sliced or indexed, and must use only static or dynamic indices. However,
 436 | ||| across axes, we can mix these four arbitrarily. For example, with `slice [2.to 4, at 1] x` to
 437 | ||| get
 438 | ||| ```
 439 | ||| x : Tensor [2] S32
 440 | ||| x = tensor [13, 19]
 441 | ||| ```
 442 | ||| or with `i = tensor 2` in `slice [at 1, i.size 2] x` to get
 443 | ||| ```
 444 | ||| x : Tensor [2] S32
 445 | ||| x = tensor [7, 8]
 446 | ||| ```
 447 | |||
 448 | ||| Slices and indices apply to the leading axes of the tensor. For trailing axes omitted from the
