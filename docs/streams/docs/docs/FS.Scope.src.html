 17 | -- Interrupt
 18 | --------------------------------------------------------------------------------
 19 |
 20 | ||| An interruption handler.
 21 | |||
 22 | ||| This is used for cross-stream interruption when running streams in
 23 | ||| parallel. For instance, when nondeterministically merging two streams
 24 | ||| A and B (see `FS.Concurrent.merge`), A and B will be run in parallel
 25 | ||| each on its own fiber. Both should be interrupted if downstream terminates,
 26 | ||| for instance because the number of emitted values have been limited via
 27 | ||| a call to `take`.
 28 | |||
 29 | ||| Implementation detail: When running a stream, we check on each iteration
 30 | ||| whether the current scope has been interrupted or not. In addition, in
 31 | ||| case of wrapped effectful computations - which might be potentially long
 32 | ||| running (think of a timer or waiting for a connection or a mouse click) -
 33 | ||| the wrapped effect is raced against the stream being interrupted.

 36 |   None : Interrupt f
 37 |   I    : (def : Deferred World a) -> Interrupt (Async e)

 64 | ||| Cancelation scopes
 65 | |||
 66 | ||| Functional streams are evaluated in scopes, which are organized as
 67 | ||| a tree, just like stream evaluation can be thought of as a tree:
 68 | ||| Sequencing of streams means nesting in the resulting scope tree,
 69 | ||| while parallel evaluation (for instance, when zipping or merging
 70 | ||| streams) means branching. Once a stream is exhausted,
 71 | ||| it's scope is cleaned up and all resources allocated in this scope
 72 | ||| are released, including the resources of all child scope.
 73 | |||
 74 | ||| In addition to (internal) cancelation, streams can be run concurrently,
 75 | ||| in which case the can be interrupted by an external event such as
 76 | ||| the exhaustion of a timer or the termination of another stream.
 77 | ||| At every evaluation step of a stream we check, if the current scope
 78 | ||| has been canceled. If this is the case, evaluation of the stream
 79 | ||| is aborted.
 80 | |||
 81 | ||| Just like `Pull`s and `Stream`s, a `Scope` is parameterized by its
 82 | ||| effect type.
