451 | ||| A command line argument of a call to the system shell.
452 | public export
453 | data CmdArg : Type where
454 |
455 |   ||| Escapable alternative represents a string that should be understood as a
456 |   ||| single argument disregarding its contents, i.e. the whole string should
457 |   ||| be passed as an argument even if it contains spaces or characters that
458 |   ||| are normally understood specially by shell.
459 |   Escapable : String -> CmdArg
460 |
461 |   ||| No escape alternative represents a raw string that is used "as is" at the
462 |   ||| shell call, thus allowing to pass commands to the shell itself.
463 |   ||| Be aware that spaces inside these strings would form several actual
464 |   ||| arguments and quotation marks and backslash symbols may interfere with
465 |   ||| escaping of other arguments.
466 |   NoEscape : String -> CmdArg
467 |
468 | ||| Interface that is used to mark which types can be used as a source for
469 | ||| command line arguments when building lists of such.
470 | ||| This allows to be able to pass different literals and expressions without
471 | ||| explicit wrapping them into the `CmdArg` type.
472 | public export
473 | interface CmdArgable a where
474 |   toCmdArg : a -> CmdArg
475 |
476 | export %inline
477 | CmdArgable CmdArg where
478 |   toCmdArg = id

481 | ||| to be used as an escapable command line argument.
482 | export %inline
483 | Interpolation a => CmdArgable a where
484 |   toCmdArg = Escapable . interpolate
485 |
486 | ||| A list of command line arguments.
487 | |||
488 | ||| This type is meant to look syntacitcally as a simple list, however
489 | ||| containing possibly values of different types which can form a command
490 | ||| line argument.
491 | |||
492 | ||| For example, a call to some command with redirection may look like this:
493 | ||| `sys ["echo", "a", NoEscape ">", file]`,
494 | ||| when, say, `file` is a value of type `File Abs`.

503 |
504 | ||| Converts a list of command line arguments to a single string
505 | ||| while putting spaces between arguments and escaping appropriate ones.
506 | |||
507 | ||| For example, call `escapeCmd ["echo", "&&", NoEscape "&&", "echo", "yes"]`

520 |   |||

522 |   ||| whether they should be escaped or not.
523 |   ||| This allows, say, to form an argument which partially contains something
