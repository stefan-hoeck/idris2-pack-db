47 | ||| With an ordinary container `c`, the Pi and Sigma type simple are the 
48 | ||| dependent function ((s : c.Shp) -> c.Pos s) and the dependent pair
49 | ||| ((s : c.Shp ** c.Pos s)) type: they rely on the (co)product in Set.
50 | ||| When the container is additive, the Pi and Sigma type rely on the 
51 | ||| (co)product in the category ComMon. Here Pi stays the same, but Sigma
52 | ||| ends up being a subtype of the Pi type, with finite support. This means that
53 | ||| in the finitary case, product and coproduct coincide.
54 | |||
55 | ||| This is a complicated way of saying something simple:
56 | ||| The Sigma type, as inherited from Set, is not a monoid. This is because,
57 | ||| despite the fact that `c` gives us a monoid structure on every `c.Pos s`, we
58 | ||| still can't add `(s1 ** p1)` and `(s2 ** p2)` when `s1 ≠ s2` as
59 | ||| `p1` and `p2` have different types. At best, we could do it if `c.Shp` was
60 | ||| a monoid, and `c.Pos` was somehow laxly preserving the monoid structure.
61 | |||
62 | ||| Instead, we need to use the Pi type representation: ((s : c.Shp) -> c.Pos s)
63 | ||| whose monoid structure is given pointwise. When `c.Shp` is an infinite type,
64 | ||| we need to ensure that the map above has finite support. Carrying this 
65 | ||| explicit support data together with the function is very fiddly
66 | |||
67 | ||| It turns out that there is a pragmatic representation of the coproduct:
68 | ||| simply as a list of pairs `(s, p)` where `p : c.Pos s` such that:
69 | ||| 1) The list order doesn't matter (we need to quotient it out by permutation)
70 | ||| 2) Pairs where output is zero can be dropped, i.e. `(s, 0) : xs = xs`
71 | ||| 3) Same-shape entires add: `(s, p1) : (s, p2) : xs` = (s, p1 + p2) : xs`
72 | ||| That is, all maps that consume this type have to preserve these properties.
73 | |||
74 | ||| It turns out that this works surprisingly well, and helps us be performant
75 | ||| especially when dealing with autodiff.
76 | |||
77 | ||| In other words, any dependent pairs that want to be a monoid should ask
78 | ||| themselves if they're instead a list of input-output pairs.
